变量提升

在浏览器执行代码前，解析器会先分析代码：

1var web = "hello";
2console.log(web);
3var class = "world"

上述代码在执行时会直接报错，因为解析器在执行代码前分析时发现了问题；

 1console.log(web);
 2var web = "hello.com";
 3
 4//上述代码相当于
 5console.log(web);
 6var web ;
 7web = "hello.com";
 8
 9//变量提升
10var web ;
11console.log(web);
12web = "hello.com";

解析器在分析时，会将变量声明的语句提升，因此执行时会输出undefined；

 1function hd() {

 3    var web = "hello.com";
 4  }
 5  console.log(web);
 6}
 7hd();
 8
 9//变量提升
10function hd() {
11  var web;
12  if(false) {
13    web = "hello.com";
14  }
15  console.log(web);
16}

块作用域与全局污染

全局污染

1function show() {
2  web = "hello.com";
3  console.log(web);
4};
5show();
6console.log(web);

上述代码中web并没有通过var，let，const声明，但仍可以使用；然而web应该只在show()函数中被使用，但此时在函数外也能使用该变量，这就造成了全局污染。

使用严格模式

1"use strict"
2function show() {
3  web = "hello.com";
4  console.log(web);
5};
6show();
7console.log(web);


1"use strict"
2function show() {
3  let web = "hello.com";
4  console.log(web);
5};
6show();
7console.log(web);

在js代码中添加"use strict"语句表明使用严格模式，要求变量必须先声明再使用。

块作用域

var声明的变量只有函数作用域和全局作用域：

1var i = 99;
2
3for (var i = 0; i < 5; i++) {
4  console.log("循环内的i：",i);
5};
6console.log("循环外的i：", i);

for循环内声明的i影响到了for循环外的i，造成了污染；

 1var i = 99;
 2for (let i = 0; i < 5; i++) {
 3  console.log("循环内的i：",i);
 4};
 5console.log("循环外的i：", i);
 6
 7let i = 99;
 8for (let i = 0; i < 5; i++) {
 9  console.log("循环内的i：",i);
10};
11console.log("循环外的i：", i);

let声明的变量拥有块作用域，因此for循环内的i不会影响到for循环外的i。

对于var拥有函数作用域的进一步解释

1(function(){
2  var $ = (window.$ = {});
3  $.web = "hello.com";
4  var url = "nihao.com";
5}.bind(window)());//这是一个立即执行函数
6
7console.log(url);

在上述代码中，用var声明的变量$被定义在一个立即执行函数中，由于var拥有函数作用域，因此在这个立即执行函数外无法访问到$；通过在该立即执行函数内声明getUrl()和get$()子函数，可以将在该立即执行函数中的声明的变量暴露给外界使用。

 1(function(){
 2  var $ = (window.$ = {});
 3  $.web = "hello.com";
 4  var url = "nihao.com";
 5  $.getUrl = function() {
 6    return url;
 7  };
 8  $.get$ = function() {
 9    return $;
10  };
11}.bind(window)());
12
13console.log($.getUrl());
14console.log($.get$());

由于let拥有块作用域，因此上述代码又可以改写为：

1{
2  let $ = (window.$ = {});
3  $.web = "hello.com";
4  let url = "nihao.com";
5  };
6}//变量$, url都在一个用{}包起来的代码块中
7
8console.log(url);


 1{
 2  let $ = (window.$ = {});
 3  $.web = "hello.com";
 4  let url = "nihao.com";
 5  $.getUrl = function() {
 6    return url;
 7  };
 8  $.get$ = function() {
 9    return $;
10  };
11}
12
13console.log($.getUrl());
14console.log($.get$());

const常量声明

const也拥有块作用域，在同一作用域里，const声明的常量无法更改:

1const URL = "hello.com";
2{
3  const URL = "world.com"
4  console.log(URL);
5}
6console.log(URL);


1const URL = "hello.com";
2URL = "world.com";
3console.log(URL);


1const URL = "hello.com";
2function show() {
3  const URL = "world.com";
4  console.log(URL);
5}
6show();
7console.log(URL);

当const声明的常量是引用类型时，可以进行更改：

1const CONFIG = {
2  URL: "hello.com"
3};
4console.log(CONFIG);
5CONFIG.URL = "world.com";
6console.log(CONFIG);

对造成上述结果的分析：

使用const声明一个基本类型的常量时，当改变该常量的值时，会造成该常量引用的地址发生改变，在上述代码中，“hello.com"和"world.com"在计算机中存储的地址不同，因此再次赋值会导致URL所引用的地址发生改变，因此报错；但是声明的是一个引用类型的常量时，由于更改CONFIG.URL并不会导致CONFIG所引用的地址发生改变，因此不会报错。

对var与let的进一步说明

1var web0 = "hello.com";
2console.log(window.web0);
3
4let web1 = "world.com";
5console.log(window.web1);
6console.log(web1);

由var声明的变量会保存到全局对象window中，而let不会；且在同一作用域下，var出现重复声明不会报错，但let和const会报错。

变量冻结

使用Object.freeze()可以使由const声明的引用类型的常量不发生改变，在严格模式下，对被Object.freeze()冻结的变量进行修改会报错。

1 "use strict"
2const CONFIG = {
3  URL: "hello.com"
4};
5console.log(CONFIG);
6Object.freeze(CONFIG);
7CONFIG.URL = "world.com";
8console.log(CONFIG);

null与undefined

两者都表示没有值

null：当声明了一个引用类型变量，但没有赋值时，此时该引用类型变量为null

undefin：当声明了一个基本类型变量，但没有赋值时，此时该基本类型变量为null

在js函数中，当传递的参数，或着返回值没有值时，默认都是undefined

1function show(name){
2  console.log(name);
3};
4console.log(show());

use strict 严格模式

严格模式的作用域包含当前作用域及其子作用域：

 1function show(){
 2  "use strict"
 3  web = "hello.com";
 4  console.log(web);
 5};
 6function hd(){
 7  site = "world.com";
 8  console.log(site);
 9};
10hd();
11show();


 1function show(){
 2  web = "hello.com";
 3  console.log(web);
 4  function handle(){
 5    "use strict"
 6    url = "nihao.com";
 7    console.log(url);
 8  };
 9  handle();
10};
11function hd(){
12  site = "world.com";
13  console.log(site);
14};
15hd();
16show();

推荐在编写js代码时添加严格模式，使代码在更多的环境中运行。
