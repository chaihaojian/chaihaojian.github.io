C++提高

模板

  • C++的另一种编程思想称为范型编程，主要利用的技术就是模板
  • C++提供两种模板机制：函数模板与类模板

概念

函数模板

基本用法

函数模板的作用：

建立一个通用函数，其函数返回值类型和形参类型可以不具体指定，用一个虚拟的类型来代表

语法：

1template<typename T>//!!!不加分号
2函数声明或定义

 1//函数模板
 2//考虑写一个两数交换的函数
 3//两个函数的区别只在于参数的类型 逻辑则是相同的
 4// void swapInt(int &a, int &b)
 5// {
 6//     int temp = a;
 7//     a = b;
 8//     b = temp;
 9// };
10// void swapDouble(double &a, double &b)
11// {
12//     double temp = a;
13//     a = b;
14//     b = temp;
15// };
16
17//通过声明一个模版函数 让一个函数满足多种参数类型
18template<typename T> //表明T是一个未确定的数据类型
19void mySwap(T &a, T &b)
20{
21    T temp = a;
22    a = b;
23    b = temp;
24};
25
26int main()
27{
28    int a = 10;
29    int b = 20;
30    double c = 1.1;
31    double d = 2.2;
32    
33    //通过自动类型推导的方式使用模版函数
34    mySwap(a, b);
35    cout << "a = " << a << endl;
36    cout << "b = " << b << endl;
37
38    //通过显式声明参数类型使用模版函数
39    mySwap<double>(c, d);
40    cout << "c = " << c << endl;
41    cout << "d = " << d << endl;
42
43    return 0;
44};

注意：

  • 自动类型推导，必须推导出一致的数据类型T，才可以使用
  • 模版必须要确定出T的数据类型，才可以使用

函数模版案例

 1//函数模版的案例 
 2//一个通用的选择排序模版
 3template<typename T>
 4void mySwap(T &a, T &b)
 5{
 6    T temp = a;
 7    a = b; 
 8    b = temp;
 9};
10
11template<typename T>
12void myPrint(T arr[], int len)
13{
14    for (int i = 0; i < len; i++)
15    {
16        cout << arr[i] << " ";
17    };
18    cout << endl;
19};
20
21template<typename T>
22void mySort(T arr[], int len)
23{
24    for (int i = 0; i < len; i++)
25    {
26        int maxIdx = i;
27        for (int j = i + 1; j < len; j++)
28        {
29            if (arr[maxIdx] < arr[j])
30            {
31                maxIdx = j;
32            };
33        };
34        if (maxIdx != i)
35        {
36            mySwap(arr[i], arr[maxIdx]);
37        };
38    };
39};
40
41int main()

51    return 0;
52};

普通函数与模版函数的区别

  • 普通函数调用时可以发生自动类型转换（隐式类型转换）
  • 函数模版调用时，如果利用自动类型推导，不会发生隐式类型转换
  • 如果模版函数调用时显式指定了参数类型，则可以发生类型转换
 1//普通函数与模版函数的区别
 2int myAdd01(int a, int b)
 3{
 4    return a + b;
 5};
 6
 7template<typename T>
 8T myAdd02(T a, T b)
 9{
10    return a + b;
11};
12
13int main()
14{
15    int a = 10;
16    int b = 20;
17    char c = 'c';
18
19    cout << "myAdd01(a + b) = " << myAdd01(a, b) << endl;
20    cout << "myAdd01(a + c) = " << myAdd01(a, c) << endl;
21
22    cout << "myAdd02(a + b) = " << myAdd02(a, b) << endl;
23    //发生自动类型推导时 无法进行类型转换
24    //cout << "myAdd02(a + c) = " << myAdd02(a, c) << endl;
25    cout << "myAdd02(a + c) = " << myAdd02<int>(a, c) << endl;
26    return 0;
27};

普通函数与函数模版的调用规则

调用规则：

  1. 如果函数模版和普通函数都可以调用，优先调用普通函数
  2. 可以通过空模版参数列表来强制调用函数模版
  3. 函数模版可以发生重载
  4. 如果函数模版可以产生更好的匹配，则优先调用函数模版
 1//普通函数与函数模版的调用规则
 2void myPrint(int a, int b);
 3void myPrint(int a, int b)
 4{
 5    cout << "普通函数调用" << endl;
 6};
 7
 8template<typename T>
 9void myPrint(T a, T b)
10{
11    cout << "函数模版调用" << endl;
12};
13
14//函数模版可以发生重载
15template<typename T>
16void myPrint(T a, T b, T c)
17{
18    cout << "重载的函数模版调用" << endl;
19};
20
21int main()
22{
23    int a = 10;
24    int b = 20;
25    char c = 'c';
26    char d = 'd';
27    //此时调用的是普通函数
28    //但是当普通函数只是声明没有具体实现时，会报错
29    myPrint(a, b);
30
31    //通过空模版参数列表强制调用函数模版
32    myPrint<>(a, b);
33    //调用重载的函数模版
34    myPrint(a, b, 100);
35
36    //此时函数模版可以进行自动类型推导 推导出T为char类型
37    //而如果调用普通函数 则需要发生类型转换
38    //因此编译器认为函数模版产生了更好的匹配
39    myPrint(c, d);
40    return 0;
41}

模版的局限性

模版在一些特定的情况下无法正常使用：

1template<typename T>
2void func(T a, Tb)
3{
4    a = b;
5    if (a > b)
6    {
7        ...
8    };
9};

当 a 与 b 是数组时，无法进行赋值，或者当 a 与 b 是自定义数据类型时，无法进行比较

因此，C++为了解决这种问题，提供模版的重载，可以**为这些特定的类型提供具体化的模版 **

 1//模版的局限性
 2class Person
 3{
 4public:
 5    string Name;
 6    int Age;
 7    Person(string n, int a)
 8    {
 9        Name = n;
10        Age = a;
11    };
12};
13
14template<typename T>
15bool isEqual(T a, T b)
16{
17    if (a == b)
18    {
19        return true;
20    };
21    return false;
22};
23
24//重载一个特定的能够比较Person类的isEqual函数
25bool isEqual(Person &p1, Person &p2)
26{
27    if (p1.Name == p2.Name && p1.Age == p2.Age)
28    {
29        return true;
30    };
31    return false;
32};
33
34int main()
35{
36    int a = 10;
37    int b = 20;
38    Person p1("Tom", 18);
39    Person p2("Jim", 18);
40    cout << isEqual(a, b) << endl;
41    cout << isEqual(p1, p2) << endl;
42    return 0;
43};

类模板

基本语法

类模板的作用：

  • 建立一个通用类，类中的成员数据类型可以不具体指定，用一个虚拟的类型来代表

语法：

1template<class T>
2类

 1//类模版基本用法
 2template<class NameType, class AgeType>
 3class Person
 4{
 5public:
 6    NameType Name;
 7    AgeType Age;
 8    Person(NameType n, AgeType a)
 9    {
10        this->Name = n;
11        this->Age = a;
12    };
13    void Show()
14    {
15        cout << "Name: " << this->Name << endl << "Age: " << this->Age << endl;
16    };
17};
18
19int main()
20{
21    Person<string, int> p("Tom", 18);
22    p.Show();
23    return 0;
24};

类模板与函数模板的区别

类模板与函数模板的区别主要有两点：

  1. 类模板没有自动类型推导的使用方式，因此必须要显式指定参数类型
  2. 类模板在模板参数列表中可以有默认参数
 1//类模板与函数模板的区别
 2//template<class NameType, class AgeType>
 3//类模板在声明时可以在参数列表中指定默认的参数类型
 4template<class NameType = string, class AgeType = int>
 5class Person
 6{
 7public:
 8    NameType Name;
 9    AgeType Age;
10    Person(NameType n, AgeType a)
11    {
12        this->Name = n;
13        this->Age = a;
14    };
15    void Show()
16    {
17        cout << "Name: " << this->Name << endl << "Age: " << this->Age << endl;
18    };
19};
20
21int main()
22{
23    //如果有默认参数类型 则可以不指定参数类型
24    //如果没有默认参数类型 由于类模板无法进行类型推导 因此会报错
25    Person<> p("Tom", 18);
26    p.Show();
27    return 0;
28};

类模板中成员函数的创建时机

类模板中成员函数和普通类中成员函数的创建时机是有区别的：

  • 普通类中的成员函数一开始就可以创建
  • 类模板中的成员函数在调用时才创建
 1//类模板中成员函数的创建时机
 2class Person
 3{
 4public:
 5    void show()
 6    {

 8    };
 9};

17    {
18        obj.show();
19    };
20};
21
22int main()
23{
24    //myPerson<Person> p;
25    
26    return 0;
27};

上述代码可以编译成功，obj为未知类型P，上述代码中没有调用obj.show()，且编译成功，因此证明类模板中的成员函数时在调用时才创建

类模板对象做函数参数

当类模板实例作为参数向函数传递时，一共有三种方法：

  1. 指定传入的类型（最常用）
  2. 参数模板化
  3. 整个类模板化
 1//类模板对象做函数参数
 2template<class NameType, class AgeType>
 3class Person
 4{
 5public:
 6    NameType Name;
 7    AgeType Age;
 8    Person(NameType n, AgeType a)
 9    {
10        this->Name = n;
11        this->Age = a;
12    };
13    void Show()
14    {
15        cout << "Name: " << this->Name << " Age: " << this->Age <<endl;
16    };
17};
18
19//方式一：指定传入的类型（最常用）
20void ShowPerson1(Person<string, int> &p)
21{
22    p.Show();
23};
24
25//方式二：参数模板化（类模板与函数模板配合使用）
26template<typename NameType, typename AgeType>
27void ShowPerson2(Person<NameType, AgeType> &p)
28{
29    p.Show();
30};
31
32//方式三：将整个类模板化（类模板与函数模板配合使用）
33template<typename PersonType>
34void ShowPerson3(PersonType &p)
35{
36    p.Show();
37};
38
39
40int main()
41{
42    Person<string, int> p("Tom", 18);
43    ShowPerson1(p);
44    ShowPerson2(p);
45    ShowPerson3(p);
46    return 0;
47};

类模板与继承

当类模板碰到继承时，需要注意以下几点：

  • 当子类继承的父类是一个类模板时，子类在声明的时候，要指定父类中T的类型
  • 如果不指定，编译器无法给子类分配内存
  • 如果想要灵活的指定父类中T的类型，则需要将子类也声明成类模板
 1//类模板与继承
 2template<class T>
 3class Base
 4{
 5public:
 6    T arg1;
 7};
 8
 9//子类在继承一个类模板时，需要指定父类中T的类型
10class Son1 : public Base<int>
11{
12public:
13    int arg2;
14};
15
16//或者可以将子类也声明为一个类模板
17//但仍需要指定将哪个类型参数作为父类中T的类型参数
18template<class T1, class T2>
19class Son2 : public Base<T2> //声明中的T2将作为父类中T的类型
20{
21public:
22    T1 arg3;
23};
24
25int main()
26{
27    Son1 s1;
28    Son2<int, char> s2;
29    return 0;
30};

类模板中成员函数在类外实现时，需要加上模板参数列表

21template<typename NameType, typename AgeType>
22Person<NameType, AgeType>::Person(NameType n, AgeType a)

类模板分文件编写

问题：

  • 类模板中的成员函数是在调用阶段再创建的，因此会导致分文件编写时链接不到
 1#include <iostream>
 2#include <string>
 3#include <unistd.h>
 4using namespace std;
 5
 6template<class NameType, class AgeType>
 7class Person
 8{
 9public:
10    NameType Name;
11    AgeType Age;
12    Person(NameType n, AgeType a);
13    void Show();
14};
15
16template<typename NameType, typename AgeType>
17Person<NameType, AgeType>::Person(NameType n, AgeType a)
18{
19    this->Name = n;
20    this->Age = a;
21};
22
23template<typename NameType, typename AgeType>
24void Person<NameType, AgeType>::Show()
25{
26    cout << "Name: " << this->Name << " Age: " << this->Age << endl;
27};
28
29int main()
30{
31    Person<string, int> p("Tom", 18);
32    p.Show();
33    return 0;
34};

上述代码可以分文件编写为：

 8{
 9public:
10    NameType Name;
11    AgeType Age;
12    Person(NameType n, AgeType a);
13    void Show();
14};

./src/chap3/person.cpp

 2
 3template<typename NameType, typename AgeType>

 5{
 6    this->Name = n;
 7    this->Age = a;
 8};
 9

./src/chap3/main.cpp

 9    //将下面两行代码注释掉 会成功完成编译

编译上述3个文件：

clang++ main.cpp person.cpp -o ../../target/main

会产生如下报错：

产生错误的原因分析：

  1. 预处理阶段，main.cpp 中的 #include "../../inc/person.h"会被对应的 person.h文件替换，

    同时person.cpp 中的 #include "../../inc/person.h"也会被 person.h文件替换，

    生成main.i和person.i文件

  2. 编译阶段，将main.cpp和person.cpp分别编译成汇编代码文件，

    生成main.s和person.s文件

  3. 汇编阶段，将main.cpp和person.cpp对应的汇编代码文件转化为二进制文件，

    生成main.o和person.o文件

  4. 链接阶段，将汇编阶段产生的两个二进制文件合并，找到各个目标文件中定义的函数或变量的地址，

    具体到上述案例中则为：链接器需要在person.o中找到main.o中Person<string, int> p("Tom", 18)的具体实现，即在person.o中找到Person<string, int>这一具体的类所对应的构造函数，而事实上，由于在person.cpp文件中，并没有实例化Person<string, int>这一个具体的类，因此也不会实例化该类对应的构造函数，因此链接器在person.o中无法找到main.o中Person<string, int> p("Tom", 18)的具体实现，void Person<string, int>::show()也是同理。

总结：类模板的分文件编写，导致具体声明的模板类没有完成实例化，从而导致链接错误。

解决方式：

  • 方式一：直接包含.cpp源文件

     1#include <iostream>
 2#include <string>
 3#include "person.cpp"
 4using namespace std;
 5

 7{
 8    Person<string, int> p("Tom", 18);
 9    p.Show();
10    return 0;
11};

  • 方式二：将声明和实现写到同一个文件中，并更改后缀名为.hpp，.hpp是约定的名称，并不是强制
